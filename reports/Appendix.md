# Appendix

## Appendix A: Installation and Setup Guide

To deploy and run the **QuillStack AI** platform locally, the following environment setup and installation procedures must be followed:

### A.1 Prerequisites
*   **Node.js**: Version 18.0.0 or higher.
*   **MongoDB**: Local installation or a MongoDB Atlas cloud cluster URI.
*   **Package Manager**: NPM (Node Package Manager).
*   **API Key**: A valid Google Gemini API key from Google AI Studio.

### A.2 Installation Steps
1.  **Clone the Repository**:
    ```bash
    git clone https://github.com/user/quillstack-ai.git
    cd quillstack-ai
    ```
2.  **Backend Configuration**:
    *   Navigate to the `backend` directory.
    *   Create a `.env` file and populate it with the following keys:
        ```env
        PORT=5000
        MONGODB_URI=your_mongodb_connection_string
        JWT_SECRET=your_secure_random_secret
        GEMINI_API_KEY=your_google_gemini_api_key
        ```
    *   Install dependencies: `npm install`
3.  **Frontend Configuration**:
    *   Navigate to the `frontend` directory.
    *   Install dependencies: `npm install`
4.  **Running the Application**:
    *   Start the backend: `npm run dev` (from the backend folder).
    *   Start the frontend: `npm run dev` (from the frontend folder).
    *   Access the application at `http://localhost:3000`.

## Appendix B: System API Endpoints

The backend is structured as a RESTful API. The primary endpoints are detailed below:

| Category | Method | Endpoint | Description |
| :--- | :--- | :--- | :--- |
| **Auth** | POST | `/api/user/register` | User registration and account creation. |
| **Auth** | POST | `/api/user/login` | User authentication and JWT issuance. |
| **Docs** | POST | `/api/docs/generate` | Uploads code and generates AI documentation. |
| **Docs** | GET | `/api/docs/all` | Retrieves generation history for the logged-in user. |
| **Docs** | DELETE | `/api/docs/:id` | Securely removes a document from the database. |
| **Admin** | GET | `/api/admin/stats` | Fetches global system metrics for administrators. |
